## 2. 下一步行动 (Next Steps)

**"Stop designing. Start coding."**

### P0: Phase 4.2 - 集群适配 (Cluster Adapter)
*   **痛点**: 目前 `server` 命令虽然能启动，但仍使用旧的 Task 结构和执行逻辑，无法利用新开发的 Native Scanner 能力。
*   **目标**: 让 `neoAgent server` 真正成为集群中的 Worker，使用新的 Runner 执行 Master 下发的任务。
*   **行动**:
    1.  **重构 Server Loop**: 修改 `cmd/agent/server` 中的轮询逻辑。
    2.  **Task Adapter**: 实现 `TaskMapper`，将 Master 下发的 JSON Task 转换为内部 `model.Task`。
    3.  **Result Adapter**: 将 `Pipeline` 或 `Scanner` 的执行结果转换为 Master 期望的回传格式。
    4.  **Auth Persistence**: 完善 Token 的本地存储和自动加载。

### P1: Phase 3.6 - 高级并发优化 (Continued)
*   **目标**: 进一步压榨性能。
*   **状态**: 🟢 **部分完成 (探针调度)**
*   **行动**:
    1.  优化 `Runner` 接口，支持动态并发调整 (Adaptive Rate Limiting)。
    2.  引入 `RttEstimator` 动态调整超时时间。

*   **阻碍**: 无。

## 4. 最近更新记录 (Changelog)

### [2026-01-29] Pipeline 全流程编排实现
**新增功能**:
1.  **Pipeline 模块**: 实现了 `internal/core/pipeline`，包含 `AutoRunner` 和 `PipelineContext`。
2.  **CLI 增强**: `scan run` 支持自动化扫描，新增 `--summary` 参数输出汇总报告。
3.  **日志优化**: CLI 增加 `--log-level` 支持，优化了扫描过程中的日志干扰。
4.  **漏斗过滤**: 确立了 Host Discovery -> Port Scan -> Service Identification 的过滤机制，显著提升网段扫描效率。

### [2026-01-27] Service 扫描引擎深度优化
**优化内容**:
1.  **SoftMatch 感知 (Hint Awareness)**: 重构了指纹匹配逻辑，使引擎能识别 SoftMatch 提供的线索（如 "http", "ssl"），而不仅仅是丢弃或硬返回。
2.  **动态探针调度 (Dynamic Probe Scheduling)**: 实现了 `prioritizeProbes` 算法。当识别出服务线索时，立即将相关的高频探针（如 `GetRequest`）提升至队列头部，大幅减少无效发包。
