# 开发进度评估 (截至 2026-01-08)

基于重构方案与代码审计，当前控制面（Control Plane）的整体完成度约 **90%**：Orchestrator 已实现真正的 **DAG 并行执行**（基于 `Predecessors` 依赖解析），并且调度链路已具备 **目标解析 -> 策略拦截 -> 任务生成 -> Pull 分发 -> CAS 认领** 的完整闭环。

数据面（Data Plane）方面："左半边"（目标生成）已完成；"右半边"（结果回流与清洗）已具备 **摄入 + 队列 + Worker 框架 + Host/Service/Web 入库主链路**，但 ETL 的映射覆盖率与合并细节仍有缺口（部分 ResultType 未实现、漏洞与 WebDetail 仍未真正落库）。

### 2. 下一步开发建议

根据 "重构路线图 (Implementation Path)"，目前系统处于 **Phase 4/5 的交界**：调度与策略链路已基本就绪，但数据闭环仍需补齐 ETL 的落库细节；同时 Phase 5 中最关键的跨端能力（指纹同步）尚未落地。

**推荐核心任务：先做指纹同步，再补齐 ETL 落库缺口**

**具体开发任务清单：**

1.  **Asset Enrichment (资产丰富化 - 离线治理)**:
    *   **FingerprintMatcher Job**: 实现后台任务，定期扫描 `AssetService` 中 `Product` 为空但 `Banner` 不为空的记录，调用指纹服务进行补全。**(✅ 已完成)**
    *   **Tag Linkage**: 实现指纹识别结果自动打标签 (TagSystem) 的逻辑，支持配置是否启用此联动。**(✅ 已完成)**

2.  **Master-Agent 指纹同步机制**:
    *   **API 实现**: 当前代码中仅存在设计与 README，尚未发现可用的 Master 端版本查询与规则下载接口实现，需要补齐（建议路径以 `/api/v1/agent/fingerprint/version` 与 `/api/v1/agent/fingerprint/download` 为准）。
    *   **规则管理**: 确保 Master 端的指纹库 (DB/File) 是最新的，并维护版本号。

3.  **Tool Adapter Layer (工具适配)**:
    *   **Command Factory**: 实现 Nmap/Masscan 等工具的命令模板生成。
    *   **Output Normalizer**: 确保 Agent 上报的 JSON 格式与 Master 的 Mapper 兼容。

**决策**: 优先实现 **指纹同步 API** 以支持 Agent 的离线识别能力；随后补齐 **ETL Mapper/Merger 缺口**（漏洞与 WebDetail 真实落库），最后再完善 **工具适配层** 与 **资源调度优化**。
