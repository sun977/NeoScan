### 2. 下一步开发建议

根据 "重构路线图 (Implementation Path)"，目前系统处于 **Phase 4: The Cleaner (数据清洗与闭环)** 的收尾阶段。

**推荐核心任务：构建离线治理与指纹同步体系**

**具体开发任务清单：**

1.  **Offline Governance (离线治理)**:
    *   **FingerprintMatcher Job**: 实现后台任务，定期扫描 `AssetService` 中 `Product` 为空但 `Banner` 不为空的记录，调用指纹服务进行补全。**(基础功能)**
    *   **Tag Linkage**: 实现指纹识别结果自动打标签 (TagSystem) 的逻辑，支持配置是否启用此联动。**(可选增强)**

2.  **Master-Agent 指纹同步机制**:
    *   **API 实现**: 实现 Master 端的指纹版本查询与规则下载接口 (`/api/v1/fingerprint/version`, `/api/v1/fingerprint/download`)。
    *   **规则管理**: 确保 Master 端的指纹库 (DB/File) 是最新的，并维护版本号。

3.  **Tool Adapter Layer (工具适配)**:
    *   **Command Factory**: 实现 Nmap/Masscan 等工具的命令模板生成。
    *   **Output Normalizer**: 确保 Agent 上报的 JSON 格式与 Master 的 Mapper 兼容。

**决策**: 优先实现 **指纹同步 API** 以支持 Agent 的离线识别能力，随后开发 **离线治理 Job** 以完善数据质量。
