### 2. 下一步开发建议

根据 "重构路线图 (Implementation Path)"，目前系统处于 **Phase 4: The Cleaner (数据清洗与闭环)** 的攻坚阶段。基础框架已就绪，接下来的重点是完善具体的业务逻辑。

**推荐核心任务：完善 ETL 引擎与资产入库逻辑**

**具体开发任务清单：**

1.  **Asset ETL Engine (深度实现)**:
    *   **Merger 实现**: 完善 `AssetMerger` 的 `Merge` 方法，实现对 `AssetHost`, `AssetService`, `AssetWeb` 的幂等更新 (Upsert)。需要处理并发写入冲突和时间戳更新。
    *   **WebCrawlerDataHandler 实现**: 填充 `Handle` 方法，解析爬虫结果 JSON，提取 Web 指纹，并将截图等大文件转存到对象存储 (S3/MinIO) 或本地文件系统。
    *   **Mapper 扩展**: 补充 `mapVulnFinding`, `mapWebEndpoint` 等暂未实现的映射逻辑。

2.  **Master-Agent 指纹同步机制**:
    *   **API 实现**: 实现 Master 端的指纹版本查询与规则下载接口。
    *   **规则管理**: 确保 Master 端的指纹库 (DB/File) 是最新的，并维护版本号。

3.  **Tool Adapter Layer (工具适配)**:
    *   **Command Factory**: 实现 Nmap/Masscan 等工具的命令模板生成。
    *   **Output Normalizer**: 确保 Agent 上报的 JSON 格式与 Master 的 Mapper 兼容。

**决策**: 优先完成 **Merger** 的核心逻辑，确保扫描结果能够真正入库并可见；随后完善 **WebCrawlerDataHandler** 处理 Web 扫描结果。
