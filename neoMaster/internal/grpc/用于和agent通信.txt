## 通信协议分离
- HTTP接口 ：用户登录使用HTTP REST API ( /api/v1/auth/login )，通过浏览器或前端应用访问
- gRPC接口 ：主要用于Master与Agent节点之间的内部通信
## gRPC拦截器的作用范围
gRPC拦截器主要用于：

1. 1.
   Agent注册认证 ：验证Agent节点的合法性
2. 2.
   内部服务调用 ：Master与Agent之间的任务分发、状态同步等
3. 3.
   系统间通信 ：服务间的安全通信保障

两者使用不同的协议栈，各自有独立的认证机制
- 用户登录 → HTTP接口 + HTTP中间件
- Agent通信 → gRPC接口 + gRPC拦截器
